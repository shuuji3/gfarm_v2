gfmdhost

Name

gfmdhost -- Gfarm metadata server host maintenance command

Synopsis

gfmdhost [-l] [-P path]

gfmdhost {-c | -m} [-P path] [-p port-number] [-C cluster-name] [-t {m | c | s} ] {metadata-server-name}

gfmdhost {-d} [-P path] {metadata-server-name}

DESCRIPTION

gfmdhost is used to display and maintain the information about the metadata server redundancy feature. gfmdhost has functions which can be selected by -l¡¢-c, -m and -d options. These functions are mutually exclusive, and only one option among them can be specified. If none of them is specified, gfmd host name will be listed as the default function of this command.

FUNCTIONS

The functions of gfmdhost command are listed in following.

<none>

If no function is specified, all the registered gfmd hosts are displayed in alphabetical order.

-l

All registered gfmd hosts are listed with detail information. They are grouped by the clusters and ordered in alphabetical.

The detail information is displayed in the following order.

Connection State (+|-)

"+" means that the gfmd connected to the master gfmd. "-" means that the gfmd does not connected to the master gfmd. This column for the master gfmd is always "+".

Server Type (master|slave)

This represents whether the gfmd is master or slave.

Replication Type (sync|async|-)

"sync" means a synchronous replication. "async" means an asynchronous replication. This column for the master is always "-".

Configration of Master Candidate (c|m|s)

"m" means a default master gfmd, "c" means a master candidate one, and "s" means a gfmd as slave only.

Cluster Name

A cluster name where the gfmd belongs to. "(default)" is displayed for the default cluster.

-c

Registers the gfmd host that is specified in the argument. -C, -p, -t can be optionally specified.

-m

Modifies the gfmd host that is specified in the argument. -C, -p, -t can be optionally specified.

-d

Deletes the gfmd host that is specified in the argument. The master gfmd cannot be deleted.

-?

Prints usage.

OPTIONS

-C cluster-name

For -c / -m functions, specifies a cluster name where the gfmd host belongs to. If it is not specified, the gfmd will belong to the default cluster.

-p port-number

For -c / -m functions, specifies a port number of the gfmd. If it is not specified, 601 as the default port number is used.

-t {m|c|s}

For -c / -m functions, specifies a configuration of master candidate. "m" means a default master, "c" means a master candidate and and "s" means a gfmd as slave-only. If it is not specified, "c" is used. The gfmd configured as the default master cannot be changed to a master candidate or slave-only directly. If the gfmd configured as non default master is set to the default master by "-t m", the previous default master changes to a master candidate at same time.

-P path

This option specifies a pathname or a Gfarm URL to identify a metadata server which will be accessed by this command. If not specified, "/" is used as the default value.
